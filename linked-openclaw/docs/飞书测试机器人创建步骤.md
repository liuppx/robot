# 飞书测试机器人创建步骤

这份文档只回答一件事：

怎么在飞书开放平台里创建两个测试机器人：

- `test-coder`
- `test-issuer`

## 1. 先打开哪里

1. 打开飞书开放平台首页：

   `https://open.feishu.cn/`

2. 登录后，点击右上角 `开发者后台`。

3. 如果你已经登录开发者后台，也可以直接打开：

   `https://open.feishu.cn/app`

## 2. 创建第一个机器人：`test-coder`

1. 在开发者后台页面，点击 `创建企业自建应用`。
2. 在应用名称里填写：`test-coder`。
3. 应用描述随便写一个测试用途说明，比如：`测试环境 coder 机器人`。
4. 上传一个图标，如果暂时没有，先随便传一个占位图也可以。
5. 点 `创建`。

创建完后，先记下这两个值：

- `App ID`
- `App Secret`

这两个值后面要交给研发填到 `coder/config/coder-bot.env`。

## 3. 给 `test-coder` 开能力和权限

### 3.1 开机器人能力

1. 进入 `test-coder` 应用详情页。
2. 在左侧找到 `添加应用能力`。
3. 选择 `机器人`。
4. 点确认添加。

### 3.2 开权限

1. 左侧进入 `开发配置`。
2. 点击 `权限管理`。
3. 在权限页里搜索：`im:message.group_msg`。
4. 找到后点击开通。

`test-coder` 现在最关键的就是这个权限。没有它，机器人虽然能发消息，但读不到群讨论消息和线程消息，`/run` 这类流程会失效。

### 3.3 不用开的东西

`test-coder` 这边：

- 不需要开事件订阅
- 不需要配回调地址
- 不需要配长连接 websocket

## 4. 创建第二个机器人：`test-issuer`

1. 回到开发者后台应用列表。
2. 再点一次 `创建企业自建应用`。
3. 应用名称填写：`test-issuer`。
4. 描述写：`测试环境 issuer 机器人`。
5. 点 `创建`。

同样记下：

- `App ID`
- `App Secret`

这两个值后面要交给研发填到 `issuer/config/openclaw.json`。

## 5. 给 `test-issuer` 开能力和权限

### 5.1 开机器人能力

1. 进入 `test-issuer` 应用详情页。
2. 左侧点击 `添加应用能力`。
3. 选择 `机器人`。
4. 点确认添加。

### 5.2 开事件订阅

1. 左侧进入 `开发配置`。
2. 点击 `事件与回调`。
3. 在 `回调配置` 里找到订阅方式设置。
4. 选择 `长连接` 或 `WebSocket 长连接`。
5. 保存。

### 5.3 订阅消息事件

1. 仍然停留在 `事件与回调` 页面。
2. 找到 `添加事件` 或 `添加订阅事件`。
3. 搜索并勾选：`im.message.receive_v1`。
4. 保存。

`test-issuer` 不开这个事件，就收不到群里 `@机器人` 的消息。

### 5.4 权限建议

`test-issuer` 至少要保证有“接收群消息事件”所需权限。

如果你们飞书后台里能单独看到群消息权限，也建议一起开上。测试时如果发现：

- `@test-issuer` 完全没反应
- 或只有部分群消息能触发

那就再补开群消息相关权限；必要时补开：

- `im:message.group_msg`

## 6. 把机器人加进测试群

两个机器人创建完后，还要进群，否则不会在测试群里出现。

### 6.1 把 `test-coder` 拉进群

1. 打开飞书客户端。
2. 进入你准备给 `test-coder` 用的测试 handoff 群。
3. 点击右上角群设置。
4. 找到类似 `群机器人`、`群应用`、`添加机器人` 或 `添加应用` 的入口。
5. 搜索 `test-coder`。
6. 点击添加。

### 6.2 把 `test-issuer` 拉进群

1. 进入你准备给 `test-issuer` 用的测试群。
2. 打开群设置。
3. 找到 `群机器人` / `群应用` / `添加应用`。
4. 搜索 `test-issuer`。
5. 点击添加。

## 7. 还要拿到什么信息

机器人加群后，还需要把群的 `chat_id` 交给研发。

如果你不知道群 `chat_id` 怎么拿：

1. 回到飞书开放平台文档。
2. 使用“获取用户或机器人所在的群列表”接口查。
3. 在返回结果里找到对应测试群的 `chat_id`。

研发侧分别会用到：

- `test-coder`
  - `FEISHU_APP_ID`
  - `FEISHU_APP_SECRET`
  - `FEISHU_HANDOFF_CHAT_ID` 或 `FEISHU_HANDOFF_CHAT_IDS`
- `test-issuer`
  - `appId`
  - `appSecret`
  - `groupAllowFrom`

## 8. 最后一步：发布/生效

如果你在后台改完能力、权限、事件后发现群里还是搜不到机器人，或者研发说配置后不生效，再检查一次：

1. 左侧有没有 `版本管理与发布`、`应用发布` 之类入口。
2. 如果有，就创建一个版本。
3. 按页面提示提交发布。
4. 如果页面要求企业管理员审批，就让管理员完成审批。

## 9. 直接发给同事的极简版

```text
请在飞书开放平台创建两个企业自建应用：

1. test-coder
- 开机器人能力
- 开权限 im:message.group_msg
- 不需要事件订阅
- 加进测试 handoff 群
- 给我 App ID、App Secret、群 chat_id

2. test-issuer
- 开机器人能力
- 开事件与回调，订阅方式选长连接 / websocket
- 订阅 im.message.receive_v1
- 加进测试群
- 如果测试时消息接收不稳定，再补群消息相关权限，必要时补 im:message.group_msg
- 给我 App ID、App Secret、群 chat_id
```
