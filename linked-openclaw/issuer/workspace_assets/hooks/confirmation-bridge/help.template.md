Issuer 使用说明

这个机器人可以帮你做 4 件事：

- 创建，修改或关闭 Issue
- 给指定 Issue 追加评论

用户按这个流程用就行：

1. 在群里 `@issuer`，直接用自然语言描述你的需求，需指定仓库名。
2. 机器人会先回复一份草案，你先检查仓库、Issue 编号、标题、正文是不是对的。
3. 确认无误后发送 `/confirm` 提交；如果同一个群里同时有多个草案，就发送 `/confirm 仓库别名` 或 `/confirm draft:xxxx`。
4. 如果内容不对，发送 `/cancel` 取消，本次不会提交到 GitHub。

常见示例：

- 创建：`@issuer 在 robot 创建 issue，标题是“登录页接口超时”，内容是“需要补重试、超时日志和告警”`
- 更新：`@issuer 修改 robot #123，把标题改成“登录接口偶发超时”，正文补充复现步骤，跟进人改成张三`
- 关闭：`@issuer 关闭 robot #123”`
- 评论：`@issuer 给 robot #123 评论：“请测试同学今天回归验证”`

指定跟进人 / assignees：

1. 创建或更新 Issue 时，可以直接说“跟进人是张三”或“指派给 kobofare”。
2. 系统会先把这个人写进正文里的 `跟进人：姓名`。
3. 如果系统里已经配置了这个人的 GitHub 映射，会自动同步成 GitHub 的 `assignee`。
4. 如果没有配置映射，Issue 里仍然会保留 `跟进人：姓名`，但 GitHub 的 `assignee` 可能不会自动带上。

上传附件的方法：

1. 提需求时，把文字和图片/文件发在当前飞书会话里。
2. 机器人出草案时会先记录附件；只有你发送 `/confirm` 后，系统才会真正上传附件并写进 GitHub。
3. 图片会显示在 GitHub 正文里，非图片文件会显示成可点击的文件链接。
4. 默认单个附件不要超过 5MB，太大的文件可能上传失败。

使用提醒：

- 机器人默认不会直接执行，而是先给你草案。
- `/confirm` 和 `/cancel` 默认只有草案发起人本人可以操作。
- 同一个人如果在同一个仓库重复提需求，新草案会覆盖旧草案。
