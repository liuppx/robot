# 部署手册

## 1. 前置条件

- Python `3.11`
- `uv`
- `git`
- `gh`
- `codex`
- `claude`
- 一套可用的 GitHub App 凭据
- 一套可用的飞书应用凭据

说明：

- 当前系统不再依赖 OpenClaw、gateway、插件注册或运行时配置生成。
- `doctor` 会同时检查 `codex` 和 `claude` 两个 CLI。

## 2. 初始化代码

```bash
uv sync --frozen
cp config/coder-bot.env.template config/coder-bot.env
```

## 3. 准备密钥

至少准备这些文件：

- `secrets/github-app.pem`
- `secrets/github-push-key`

要求：

- `github-app.pem` 对应 `GITHUB_APP_ID`
- `github-push-key` 必须能通过 SSH 拉取和推送目标 fork

## 4. 填写环境变量

重点字段：

- `GITHUB_APP_ID`
- `GITHUB_INSTALLATION_ID`
- `GITHUB_PRIVATE_KEY_PATH`
- `GITHUB_CLONE_SSH_KEY_PATH`
- `GITHUB_FORK_OWNER`
- `ALLOWED_REPOS`
- `FEISHU_APP_ID`
- `FEISHU_APP_SECRET`
- `FEISHU_HANDOFF_CHAT_ID` 或 `FEISHU_HANDOFF_CHAT_IDS`

模型相关：

- `EXECUTION_BACKEND`
  - 只决定默认后端
  - 可选值只有 `codex`、`claude`
- `CODEX_MODEL`
- `CLAUDE_MODEL`
- `SUPPORTED_CODEX_MODELS`
- `SUPPORTED_CLAUDE_MODELS`

说明：

- 飞书里如果显式发送 `/model gpt-5.5` 或 `/model claude-opus-4-6`，系统会自动按模型推断后端。
- `ISSUE_SESSION_PREFIX` 用于生成 issue 级会话 key，默认 `gh`。

## 5. 认证检查

`Codex` 至少要有一份可用认证：

- `~/.codex/auth.json`
- 或 `CODEX_RUNTIME_HOME/auth.json`

`Claude` CLI 也必须能在当前机器上直接运行：

```bash
codex --version
claude --version
```

## 6. 运行 doctor

```bash
uv run --frozen coder-bot --env-file config/coder-bot.env doctor
```

预期：

- GitHub App JWT 通过
- GitHub Installation Token 通过
- Feishu 配置通过
- `Codex CLI 可运行`
- `Claude CLI 可运行`

## 7. 启动服务

手动启动：

```bash
./scripts/start_bot.sh
./scripts/status_bot.sh
```

停止：

```bash
./scripts/stop_bot.sh
```

systemd 安装：

```bash
./scripts/bootstrap.sh
sudo systemctl status --no-pager coder-bot
sudo journalctl -u coder-bot -n 100 --no-pager
```

## 8. 健康检查

```bash
curl -s http://127.0.0.1:9081/health
```

重点字段：

- `status`
- `execution_backend`
- `model`
- `allowed_repos`
- `queue`

## 9. 飞书冒烟测试

1. 在 handoff 群发送 `/issue <repo>`
2. 选择一个 issue：`/issue <repo> #<number> gpt-5.5`
3. 在线程里发送 `/model claude-opus-4-6`
4. 再发送 `/run`
5. 确认线程里出现排队、执行进度、结果和 PR 链接

## 10. 常见问题

### 10.1 `doctor` 提示 `CODEX_SOURCE_HOME 或 CODEX_RUNTIME_HOME 下至少要有一份可用的 auth.json`

处理：

- 检查 `~/.codex/auth.json` 是否存在
- 或把认证文件放到 `CODEX_RUNTIME_HOME/auth.json`

### 10.2 `doctor` 提示 `Claude CLI 可运行` 失败

处理：

- 确认 `CLAUDE_BIN` 指向正确的可执行文件
- 直接执行 `claude --version`
- 确认当前服务用户也能执行该命令

### 10.3 飞书命令没有反应

优先检查：

- `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 是否正确
- `FEISHU_HANDOFF_CHAT_ID(S)` 是否包含当前群
- `curl -s http://127.0.0.1:9081/health`
- `tail -f data/logs/gunicorn.error.log`

### 10.4 Git 拉仓库超时

优先检查：

- `GITHUB_CLONE_SSH_KEY_PATH` 是否可读
- 机器到 GitHub 的 SSH/HTTPS 链路是否可用
- 是否有本机 SSH 代理配置污染了 `git` 拉取链路
